最好使用自带的 git bash 工具

提交注释说明

feat: 添加代码
doc：文档
fix：修复bug
refact: 修改代码

常用命令

撤销本地修改

git checkout .

删除本地分支

git branch -d <branch>

当一个分支被推送并合并到远程分支后，-d 才会本地删除该分支。如果一个分支还没有被推送或者合并，那么可以使用-D强制删除它。

删除未完成的 rebase 记录

rm -fr .git/rebase-merge

新建分支并切换

git checkout -b cache-test02

创建远程分支

git checkout -b my-test  //在当前分支下创建my-test的本地分支分支
git push origin my-test  //将my-test分支推送到远程
git branch --set-upstream-to=origin/my-test //将本地分支my-test关联到远程分支my-test上   
git branch -a //查看远程分支

删除远程分支

git push origin --delete my-test

# 还要再删除一遍本地分支

分支问题

当有功能或者代码变动时：
1. 新建分支

2. 将多个修改合并为一个（用 git 自带的工具！！！pycharm会出现问题！！！）
git rebase -i "前一个分支的 hash"
将要合并的分支前的 pick 改为 s（即第二行开始的所有 pick 改为 s）
wq
此时若没错会弹出注释修改框
只留一个注释
wq

3. 合并到主分支上
切换分支：git checkout develop
git pull -r
git cherry-pick "合并分支的 hash"
git push

3.1 如果不想合并到主分支，只想合并分支上的提交
	git push -f 即可将提交记录改为一个

已经存在远程分支，但本地没有

拉取分支内容
git fetch origin branch-name

切换并将本地分支关联到远程分支
git checkout -b branch-name origin/branch-name

远程分支已被删除，清理本地分支

清除本地远程分支的缓存
git remote prune origin

忽略文件追踪

# 注意：只能忽略具体的某个文件
git update-index --assume-unchanged 文件路径

取消忽略
git update-index --no-assume-unchanged 文件路径

commit

撤销本次 commit

# 撤销但不删除改动和 add 操作
git reset --soft HEAD~1

# 删除并删除改动
git reset --hard HEAD~1

问题

.gitignore 文件不生效

清除缓存：git rm -r –cached .

恢复被删除的缓存：git restore –staged .
